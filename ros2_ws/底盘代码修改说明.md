# 小车底盘更换：阿克曼转向 → 四轮差速 代码修改说明

## 一、修改背景

| 项目 | 旧底盘 | 新底盘（MickX4-V3） |
|------|--------|---------------------|
| 型号 | 自研小车（阿克曼） | 米克机器人 MickX4-V3 |
| 转向方式 | 前轮转向（阿克曼），后轮驱动 | 四轮差速（轮毂电机） |
| 最小转弯半径 | >0 | 0（可原地旋转） |
| 控制器 | STM32F103 | STM32F407 |
| ROS话题 | /cmd_vel → 前轮转角+后轮速度 | /mickrobot/chassis/cmd_vel |
| 通讯方式 | 自定义帧格式（AA 55 ...） | 串口→以太网 TCP（192.168.0.7:8234） |

**需要改为**：直接转发 Twist 到 MickX4 的串口协议。

**MickX4 速度指令协议（命令字 0xF3）：**

帧格式：`AE EA 0B F3 [X速度] [Y速度] [角速度] 00 00 [校验] EF FE`

| 字节 | 含义 | 说明 |
|------|------|------|
| 0-1 | 0xAE 0xEA | 帧头 |
| 2 | 0x0B | 帧长度 |
| 3 | 0xF3 | 速度指令 |
| 4-5 | X速度(0.01m/s) | 偏移10m/s，放大100倍，uint16大端 |
| 6-7 | Y速度(0.01m/s) | 差速底盘固定为0，uint16大端 |
| 8-9 | 角速度(0.01rad/s) | 偏移10rad/s，放大100倍，uint16大端 |
| 10-11 | 0x00 0x00 | 保留 |
| 12 | 校验 | 长度+命令+所有数据，低8位 |
| 13-14 | 0xEF 0xFE | 帧尾 |

**MickX4 里程计协议（命令字 0xA7，用于解析反馈）：**

帧格式：`AE EA [长度] A7 [X速度 int16] [Y速度 int16] [角速度 int16] [校验] EF FE`

速度和角速度单位为 0.001m/s 和 0.001rad/s。

**修改建议：**

```python
def twist_to_mick_chassis_frame(self, cmd):
    """
    将 cmd_vel 转换为 MickX4 差速底盘的速度指令帧
    """
    x_speed = cmd.linear.x      # 前进速度 m/s
    y_speed = cmd.linear.y      # 侧向速度（差速底盘为0）
    angular_z = cmd.angular.z   # 旋转角速度 rad/s
    
    # 偏移量10，放大100倍
    SPEED_OFFSET = 10.0
    x_val = int((x_speed + SPEED_OFFSET) * 100)
    y_val = int((y_speed + SPEED_OFFSET) * 100)
    w_val = int((angular_z + SPEED_OFFSET) * 100)
    
    frame = bytearray(15)
    frame[0] = 0xAE
    frame[1] = 0xEA
    frame[2] = 0x0B        # 帧长度
    frame[3] = 0xF3         # 速度指令
    
    frame[4] = (x_val >> 8) & 0xFF   # X速度高位
    frame[5] = x_val & 0xFF          # X速度低位
    frame[6] = (y_val >> 8) & 0xFF   # Y速度高位
    frame[7] = y_val & 0xFF          # Y速度低位
    frame[8] = (w_val >> 8) & 0xFF   # 角速度高位
    frame[9] = w_val & 0xFF          # 角速度低位
    
    frame[10] = 0x00       # 保留
    frame[11] = 0x00       # 保留
    
    # 校验 = 长度 + 命令 + 所有数据（低8位）
    check = sum(frame[2:12]) & 0xFF
    frame[12] = check
    
    frame[13] = 0xEF
    frame[14] = 0xFE
    
    return bytes(frame)
```

### 2. **astar_planner.py** — ⚠️ 纯跟踪控制器需要修改（中等改动）

当前的纯跟踪控制器**假设阿克曼转向**：

```python
# 第878-892行（astar_planner.py）：
# 阿克曼：steer → yaw_rate = speed * tan(steer) / wheel_base
# 低速大转角时 yaw_rate 会受限

curvature = 2.0 * math.sin(alpha) / lookahead_distance
steer = math.atan(self.wheel_base * curvature)  # ← 阿克曼转角有限制
steer = max(-self.max_steer, min(self.max_steer, steer))

steering_slowdown = 1.0 - 0.65 * min(1.0, abs(steer) / self.max_steer)
goal_slowdown = max(0.25, min(1.0, goal_distance / 1.2))
speed = self.max_speed * steering_slowdown * goal_slowdown
yaw_rate = speed * math.tan(steer) / max(0.05, self.wheel_base)
```

**差速底盘的纯跟踪控制器应该改为：**

```python
# 差速底盘：直接输出 angular.z（不需要经 steer 转换）
# cmd_vel.linear.x = 速度
# cmd_vel.angular.z = 角速度

curvature = 2.0 * math.sin(alpha) / lookahead_distance

# 差速底盘：angular.z = linear.x * curvature
desired_angular = speed * curvature

# 限制最大角速度
desired_angular = max(-self.max_yaw_rate, min(self.max_yaw_rate, desired_angular))

# 不再需要 steering_slowdown（阿克曼低速时转角受限）
# 改为：大角度旋转时减速
angular_ratio = abs(desired_angular) / max(1e-6, self.max_yaw_rate)
turn_slowdown = 1.0 - 0.5 * min(1.0, angular_ratio)

goal_slowdown = max(0.25, min(1.0, goal_distance / 1.2))
speed = self.max_speed * turn_slowdown * goal_slowdown

cmd = Twist()
cmd.linear.x = speed
cmd.angular.z = desired_angular
```

**同时影响：**
- `steering_smoothing_alpha` 参数不再需要（差速底盘不需要转角平滑）
- `max_steer` 参数不再需要，替换为 `max_yaw_rate`
- `wheel_base` 参数在控制器中不再需要（但在规划中仍可用于安全半径计算）

## 四、关键注意事项

1. **话题名不同**：新底盘用 `/mickrobot/chassis/cmd_vel`，如果你不改底盘固件，需要在 `mcu_protocol_bridge.py` 中订阅 `/cmd_vel`，然后通过串口/以太网发给底盘；或者直接用米克官方的 ROS2 节点 `ros2 run mick_bringup mick_node`。

2. **串口/以太网**：MickX4 默认用 TCP（192.168.0.7:8234），旧代码是串口。`mcu_protocol_bridge.py` 需要增加 TCP 客户端功能。

3. **发布的话题**：旧底盘 MCU 发布里程计和 IMU，新底盘通过串口/以太网上报。需要实现接收解析逻辑来发布 odom。

4. **原地旋转能力**：差速底盘可以原地转弯，这意味着导航参数可以更激进（allowed turning radius = 0），规划器应该能利用这个特性。
