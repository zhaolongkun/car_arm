# 说明1：Jetson Orin Nano 与 MickX4-V3 底盘控制板的通信与控制链路

## 提问

我的上位机的 Jetson Orin Nano 怎么和我的控制板进行通信，在控制板上用什么接口进行通信，怎么给小车发送控制信号，发送给小车的控制板后，控制板又是怎么控制四个电调的，电调又是怎么控制电机的？

## 回答

---

### 完整通信链路：Jetson Orin Nano → 控制板 → 电调 → 电机

---

### ① Jetson Orin Nano ↔ 控制板 — 两种通信方式

**方式一：串口（RS232）— 默认方式**

控制板程序（`main.c` 第97行）：
```c
My_Config_USART_Init(USART1, 115200, 1);  // 串口1，波特率115200
```

在中断函数 `USART1_IRQHandler`（`stm32f4xx_it.c:241`）中，逐字节接收，收到 `0xAE 0xEA` 帧头后开始组包。包体在 `Motor_WriteData_In_Buff()`（`MOTOR_Control.c:598-727`）中解析。

**方式二：以太网 TCP（调试口）— 通过串口转以太网模块**

控制板通过 **串口转以太网模块**（USR-TCP232-T0/T2）把串口1转为 TCP。模块 IP 固定为 `192.168.0.7:8234`。Jetson 通过网线连到小车面板的"以太网调试口"，本质还是串口数据，只是物理层变成了 TCP。

> **所以你用 Jetson 控制小车，最简单的就是：**
> - Jetson USB 转 TTL 串口线 → 小车面板的 RS232 串口（DB9）
> - **或者** 网线连到面板以太网口，TCP 连 `192.168.0.7:8234`

---

### ② Jetson 发送控制信号 — 协议格式

你给控制板发命令字 **`0xF3`**（差速底盘速度模式），28 字节：

```
AE EA 0B F3 [X速度] [Y速度] [角速度] 00 00 [校验] EF FE
```

在控制板代码中是这样解析的（`MOTOR_Control.c:635-645`）：

```c
if(recived_cmd.cmd == 0xF3 && rc.sw1 == 1)  // 拨码开关在自动驾驶档
{
    // 解析速度（偏移10m/s，放大100倍）
    recived_cmd.tag_speed_x = (UART1_ReBuff[4]*256 + UART1_ReBuff[5]) / 100.0 - 10.0;
    recived_cmd.tag_speed_y = 0;
    recived_cmd.tag_speed_w = (UART1_ReBuff[8]*256 + UART1_ReBuff[9]) / 100.0 - 10.0;
    
    // 4轮差速运动学解算
    DiffX4_Wheel_Speed_Model(recived_cmd.tag_speed_x, recived_cmd.tag_speed_w);
}
```

---

### ③ 控制板 → 四个电调 — CAN 总线

关键函数 `DiffX4_Wheel_Speed_Model()`（`MOTOR_Control.c:54-85`）：

```c
void DiffX4_Wheel_Speed_Model(float speed_x, float speed_w)
{
    // 差速运动学：左右两侧速度差产生旋转
    // 参数：Chassis_Parameter_L = 0.4m（半轮距）
    v1 = speed_x + speed_w * 0.4;    // 左前轮
    v2 = v1;                           // 左后轮（和左前一样）
    v4 = -(speed_x - speed_w * 0.4);  // 右前轮（反向）
    v3 = v4;                           // 右后轮（和右前一样）
    
    // 速度(m/s) → 转速(RPM)，8寸轮胎系数 112.344668 = 60/(PI*0.17)
    tag_rpm[0] = -(112.34 * v1);   // 左前轮目标RPM
    tag_rpm[1] = -(112.34 * v2);   // 左后轮
    tag_rpm[2] = -(112.34 * v3);   // 右后轮
    tag_rpm[3] = -(112.34 * v4);   // 右前轮
}
```

然后每 10ms 定时器中断中（`MOTOR_Control.c:486-515`），通过 **CAN 总线** 向每个电调发送 RPM 指令：

```c
void ChasissDiffX4_Control_Routing(uint32_t cnt)
{
    // 急停检测
    if( Read_Isolated_Input(1) == 1) { /* 全部置零 */ }
    
    // 分时发送 4 个电机的目标转速（10ms内分4个时间片）
    switch(cnt) {
        case 1: MOTOR_APSL2DB_Set_RPM(1, tag_rpm[0]); break;  // 左前
        case 2: MOTOR_APSL2DB_Set_RPM(4, tag_rpm[3]); break;  // 右前
        case 3: MOTOR_APSL2DB_Set_RPM(3, tag_rpm[2]); break;  // 右后
        case 4: MOTOR_APSL2DB_Set_RPM(2, tag_rpm[1]); break;  // 左后
    }
}
```

CAN 总线配置（`bsp_can.c:180`）：
```c
CAN_InitStructure.CAN_Prescaler = 6;  // CAN波特率 = 1 Mbps
```

CAN 滤波器只接收 ID = `0x580` 的扩展帧（电调返回的状态帧）。

---

### ④ 电调 → 电机 — 无刷电机 FOC 控制

根据文件夹名称 `电机M3508_C620电调`，你的底盘用的是：

| 组件 | 型号 | 说明 |
|------|------|------|
| **电机**（×4） | **RoboMaster M3508** | 直流无刷减速电机，减速比 17:1，峰值扭矩 18Nm |
| **电调**（×4） | **RoboMaster C620** | 无刷电机调速器，**CAN 总线通信**，FOC 矢量控制 |

**C620 电调的工作方式：**
- 通过 **CAN 总线** 接收控制板的转速指令（RPM 目标值）
- 电调内部运行 **FOC（磁场定向控制）** 算法，把直流电转换为三相正弦波驱动无刷电机
- 电调通过 **霍尔传感器** 检测转子位置，实现闭环速度控制
- **电机编码器**（M3508 内置）返回实际转速、角度给电调，电调再通过 CAN 返回给控制板

C620 电调的 CAN ID 分配是：电机 1→0x201, 电机 2→0x202, 电机 3→0x203, 电机 4→0x204（标准 CAN 协议，每个电调有自己的 ID 应答）。

### 你的两个关键操作

**1. 给小车发控制信号：**
```bash
# 方式一：串口直连
sudo chmod 777 /dev/ttyUSB0
# 通过ROS2节点发/cmd_vel

# 方式二：以太网TCP
# 连网线到小车面板，IP设为192.168.0.201
# 连 TCP 192.168.0.7:8234
```

**2. 遥控器左上角拨码开关必须拨到最上（L档）**，表示"自动驾驶模式"，否则控制板忽略 Jetson 发来的命令（`rc.sw1 == 1` 条件）。
